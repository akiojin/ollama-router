# 作業計画（待機キュー・初期化表示対応）

## 優先度: 高
1. テスト整備
   - 全エージェント initializing 中のリクエストが ready で再開することを統合テストで確認。
   - 待機者が 1024 を超えた場合に 503 を返すことを確認。
   - initializing / ready_models フィールドのシリアライズを確認（必要ならスナップショット）。

## 優先度: 中
1. UI/UX 微調整
   - 待機件数（waiters）の表示が必要か検討し、必要なら UI / API に露出。
   - initializing / ready の表示をフィルタやソートに反映するか検討。

1. 警告整理
   - unused import / missing docs などの警告を解消。

## 優先度: 低
1. ドキュメント更新
   - 初期化中は ready 通知待ち（waiters 上限 1024）で再開し、超過時は 503 を返す挙動を README/仕様メモに追記。

2. 依存アラート対応
   - Dependabot が示す脆弱性について、必要に応じて依存アップデートを検討。
